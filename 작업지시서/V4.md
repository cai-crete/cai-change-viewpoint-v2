# 작업지시서 V4

**작성일:** 2026-04-02
**목적:** 신규 프로토콜 문서 연동 분석 및 `Integration Process.md` 개편 계획

## 1. 신규 프로토콜 문서 상세 분석
요청하신 3개의 신규 문서를 분석한 결과입니다.

| 문서명 | 주요 변경점 및 역할 |
|---|---|
| **`protocol-image to elevation-v6.md`** | 기존 v2.2에서 고도화됨. 1단장의 이미지에서 오차 없이 5면의 정사영을 추론해내는 "결정론적 BIM 컴파일러(Protocol A)"로서, '가시권 동결 $\rightarrow$ 비가시권 추론 $\rightarrow$ 앙상블 패키징(Phase 1~4)'의 강제 절차를 명시. |
| **`AEPS-v4.md`** | Protocol A(v6)의 결과물이 담기는 컨테이너 규격. 형태(Geometry)와 속성(Property)을 단독으로 쓰지 않고 항상 `ensemble_pair`로 묶어서 시각화 엔진(Protocol B)으로 넘기는 마스터 스키마 제공. |
| **`전개도작성 가이드라인.md`** | `ANALYZE` 버튼 클릭 시 발동하는 실행 가이드. 각 뷰포인트(Front, Top, Right, Left, Rear)별로 `1_Geometry_MASTER`(3D 뼈대/비례)와 `2_Property_SLAVE`(재질/광학수치)를 어떻게 정의해야 하는지 구체적인 템플릿과 기준 제시. |

## 2. `Integration Process.md` 업데이트 로직
위 분석을 바탕으로 `Integration Process.md` 파일의 다음 항목들을 전면 업데이트해야 합니다.
1. **PHASE 2의 명칭 및 참조 엔진 갱신:** 기존 `System Protocol Image to Elevation-v2.2`를 `protocol-image to elevation-v6`로 대체.
2. **AEPS-v4 스키마 반영:** Parameter Library 연동 부분에 AEPS-v4 규격과 `ensemble_pair` 1:1 매칭 철학을 강화 서술.
3. **ANALYZE 발동 순서 변경:** `ANALYZE` 클릭 연산 시 `전개도작성 가이드라인.md` 지침이 프롬프트 시스템에 추가 참전하여, 추론된 데이터 결과물이 가이드라인 문서의 스키마에 맞게 도출되도록 플로우 수정.

## 3. [판단 대기 및 3가지 제안] 생성 데이터 개발자 검증 방식
"개발자가 `전개도작성 가이드라인`을 통해 출력된 `1_Geometry_Data_MASTER`와 `2_Property_Data_SLAVE`를 확인할 수 있는 수단을 마련하라"는 요청에 대해, 프론트엔드 환경에서 데이터를 노출하는 3가지 방식을 제안합니다. **원하시는 방식 1가지를 승인해 주시면 해당 방식으로 앱 시각화 및 프로세스 연동 작업을 진행하겠습니다.**

**제안 1. 브라우저 콘솔 (Console.log) 전용 모드 (가장 가벼움)**
- 화면 UI 구조는 그대로 유지합니다.
- 대신 코어 연산을 다 마친 직후, 도출된 JSON 트리를 `console.log()`로 예쁘게 정렬하여 출력합니다.
- 일반 사용자는 볼 수 없으며, 개발자 도구(F12)를 열 수 있는 개발자만 바로 상태를 확인할 수 있는 가장 빠르고 안전한 방식입니다.

**제안 2. 옵션 패널 내 [Developer Data Inspector] 탭 추가 (UI 통합)**
- 현재 오른쪽 `Analysis Report` 패널(사이드바) UI의 탭 부분 하단에 `[ { } RAW DATA ]`라는 개발자 전용 아코디언 메뉴나 탭을 하나 더 만듭니다.
- 버튼을 누르면 AI가 추출해서 생성한 `1_Geometry_MASTER`와 `2_Property_SLAVE`의 JSON 원문 및 텍스트 데이터가 코드 하이라이팅된 뷰어로 출력되어, 편하게 마우스로 복사하고 검증할 수 있습니다.

**제안 3. 히든 단축키 호출형 오버레이 모달 (Ctrl + Shift + D)**
- 사이드바 공간을 차지하지 않도록 숨깁니다.
- 키보드 조합(예: `Ctrl + Shift + D` 또는 `~`)을 누르면 화면 전체를 덮는 투명한 검은색 터미널 오버레이(Modal) 창이 큽니다.
- 이 모달 내부에 현재 메모리에 올라와 있는 각 5면 프롬프트의 기하학/속성 데이터 상태가 실시간으로 표시됩니다.
