## 1. 현황 분석 — 비율 지시로 인한 구도 왜곡

### 1-A. 현재 방식 (V37)

```typescript
// buildPanelPrompt() — line 966, 973
- Aspect Ratio: width ${panel.wUnit} : height ${panel.hUnit}
- Fill the image to the ${panel.wUnit}:${panel.hUnit} aspect ratio
```

**예시 (W=8, D=6, H=15):**

| 패널 | 비율 지시 | AI 동작 |
|---|---|---|
| FRONT | `width 8 : height 15` | 1:1.875 세로형으로 건물 콘텐츠 강제 압축 |
| LEFT/RIGHT | `width 6 : height 15` | 1:2.5 세로형으로 강제 압축 |
| TOP | `width 8 : height 6` | 4:3 가로형으로 강제 배치 |

**근본 문제**: Gemini IMAGE_GEN 모델은 출력 캔버스가 정사각형 고정. bldgRatio 수치를 프롬프트에 명시해도 실제 캔버스 크기는 변하지 않음 → AI가 지정된 비율로 건물을 정사각형 안에 억지로 맞추려 함 → 구도 왜곡 발생.

## 2. 방안 B — 비율 지시 제거

**원칙**: 비율 수치 지시를 프롬프트에서 전면 제거. AI가 IMAGE 1(원본 사진)을 기준으로 자연스럽게 구도를 결정하도록 위임.

### 변경 내용 (line 960–980)

```diff
  ## Target
  - View: ${panel.label} elevation
- - Aspect Ratio: width ${panel.wUnit} : height ${panel.hUnit}
  - Projection: Orthographic (strictly no perspective)
  - Background: 100% transparent (alpha channel mandatory)
  - No text, no labels, no dimension lines

  ## Constraints
  - Render ONLY the ${panel.label} face of the building
- - Fill the image to the ${panel.wUnit}:${panel.hUnit} aspect ratio
  - Ground line at the bottom of the image
  - Preserve all materials, textures, and openings from IMAGE 1
  - PROHIBITED: Adding any element not visible in IMAGE 1
```

### 변경 후 동작

```
AI 입력: IMAGE 1 (원본 사진) + AEPL 파라미터
AI 판단: 원본 사진의 자연스러운 구도 그대로 참조
출력: 정사각형 캔버스, 건물 콘텐츠 자연 비율로 배치
표시: trimAndCenter() → 건물 중앙 정렬 → object-contain → 컨테이너 내 레터박스
```

### 비고

- `wUnit`, `hUnit` 필드는 PANELS 배열에서 제거하지 않음 — 아트보드 CSS 컨테이너 크기 계산에 계속 사용.
- 아트보드 레이아웃(bldgRatio 기반 컨테이너 비율)은 변경 없음.
