# 작업지시서 V33

**작성일:** 2026-04-05
**목적:** 건축물 `bldgRatio`에 따라 전개도 패널 비율 및 아트보드 크기가 자동으로 동적 산출되도록 개선 (클리핑 현상 근본 해결)

---

## 1. 현황 분석 — 클리핑의 두 가지 원인

### 원인 A: 그리드 높이 > 실제 컨테이너 높이 (V31 미해결)

| 레이어 | 높이 |
|---|---|
| 아트보드 (`height: '1200px'`) | 1200px |
| 외부 패딩 `p-6` (24px × 2) | −48px |
| 내부 패딩 `p-20` (80px × 2) | −160px |
| **실제 콘텐츠 영역 높이** | **992px** |

```
V31 계산:
  availableH = 1100 − 24 = 1076
  SCALE      = 1076 / 42 = 25.62
  그리드 높이 = 42 × 25.62 + 24 = 1100px  ← 실제 992px 대비 108px 초과
```

`flex items-center justify-center` + `overflow-auto` 조합에서
자식이 컨테이너보다 크면 플렉스 센터링이 음수 마진을 생성하여
**상단 54px(REAR)는 스크롤해도 접근 불가**, **하단 54px(FRONT)는 스크롤 필요**.

### 원인 B: 고정 슬롯 vs. 실제 건축물 비율 불일치 (V31 구조적 한계)

V31의 W:12, D:12, H:15 고정값은 `object-fit: cover`로 이미지를 표시합니다.
실제 건축물 비율(예: 광폭형 W=20, H=8)이 고정 슬롯 비율과 다를 경우
생성된 입면 이미지가 패널 안에서 강제 크롭되어 건물 양쪽이 잘립니다.

---

## 2. 변경 계획 (V33 Technical Plan)

### 핵심 방향

> V31의 고정값(W:12, D:12, H:15)을 폐기하고,
> AI 분석으로 추출된 `bldgRatio`(width, depth, height)를 패널 비율로 사용하며,
> 아트보드 크기를 그리드 크기에 맞춰 **동적으로 자동 산출**한다.
> 그리드는 항상 아트보드에 완전히 수용되어 클리핑이 발생하지 않는다.

---

### A. 동적 스케일 및 아트보드 산출 공식

```
상수 정의:
  PADDING       = 208px  ← p-6(48) + p-20(160), 가로/세로 공통
  gapTotal      = 24px   ← 12px × 2 gaps
  TARGET_SCALE  = 30     ← px/unit 목표값 (기본 가독성)
  SCALE_MIN     = 20     ← px/unit 최솟값 (초대형 건물 대응)
  MAX_ARTBOARD_W = 2400px
  MAX_ARTBOARD_H = 2000px

입력값 (bldgRatio):
  W = bldgRatio.width   (폭)
  D = bldgRatio.depth   (깊이)
  H = bldgRatio.height  (높이)
  fallback: W=10, D=8, H=15

그리드 총 유닛:
  unitsW = 2×H + W   (가로: 좌측면H + 정면W + 우측면H)
  unitsH = 2×H + D   (세로: 배면H + 평면D + 정면H)

최대 그리드 가용:
  maxGridW = 2400 − 208 − 24 = 2168px
  maxGridH = 2000 − 208 − 24 = 1768px

SCALE 산출:
  scaleByW = maxGridW / unitsW
  scaleByH = maxGridH / unitsH
  SCALE = max(SCALE_MIN, min(TARGET_SCALE, scaleByW, scaleByH))

그리드 실제 크기:
  gridW = unitsW × SCALE + gapTotal
  gridH = unitsH × SCALE + gapTotal

아트보드 크기 (자동 산출):
  artboardW = ceil(gridW + PADDING)
  artboardH = ceil(gridH + PADDING)
```

#### 검증 예시

| 건축물 타입 | W | D | H | unitsW | unitsH | SCALE | artboardW | artboardH |
|---|---|---|---|---|---|---|---|---|
| 기본형 | 10 | 8 | 15 | 40 | 38 | 30 | 1432px | 1372px |
| 정방형(V31) | 12 | 12 | 15 | 42 | 42 | 30 | 1472px | 1472px |
| 광폭형 | 20 | 10 | 10 | 40 | 30 | 30 | 1432px | 1132px |
| 광폭 초대형 | 50 | 20 | 10 | 70 | 40 | 30 | 2324px | 1432px |
| 고층형 | 8 | 8 | 40 | 88 | 88 | 20* | 1984px | 1984px |

\* SCALE_MIN=20 적용 (max artboard 제약으로 감소)

### B. 수정 위치 및 코드 (`src/App.tsx`)

#### B-1. `bldgRatio` 참조 (계산 블록 상단)

현재 `elevationImages` 렌더링 블록 안에서 `ei = item.parameters?.elevationImages`로
참조하고 있으므로, `bldgRatio`는 `item.parameters`에서 직접 가져옵니다.

```typescript
// [V33] bldgRatio 동적 참조
const bldgRatio = (item.parameters as any)?.bldgRatio as
  { width: number; depth: number; height: number } | undefined;
const W = bldgRatio?.width  ?? 10;
const D = bldgRatio?.depth  ?? 8;
const H = bldgRatio?.height ?? 15;
```

#### B-2. 스케일 및 아트보드 산출 (기존 V31 계산 블록 교체)

```typescript
// [V33] 동적 아트보드 & 스케일 산출
const gapTotal      = 24;
const PADDING       = 208;       // p-6(48) + p-20(160)
const TARGET_SCALE  = 30;
const SCALE_MIN     = 20;
const MAX_AW        = 2400;
const MAX_AH        = 2000;

const unitsW = 2 * H + W;
const unitsH = 2 * H + D;

const maxGridW = MAX_AW - PADDING - gapTotal;  // 2168
const maxGridH = MAX_AH - PADDING - gapTotal;  // 1768

const scaleByW = maxGridW / unitsW;
const scaleByH = maxGridH / unitsH;
const SCALE = Math.max(SCALE_MIN, Math.min(TARGET_SCALE, scaleByW, scaleByH));

const gridW = unitsW * SCALE + gapTotal;
const gridH = unitsH * SCALE + gapTotal;
const artboardW = Math.ceil(gridW + PADDING);
const artboardH = Math.ceil(gridH + PADDING);
```

#### B-3. 아트보드 스타일 동적 적용 (~line 1600-1606)

```typescript
// 변경 전
style={{ width: `1600px`, height: `1200px` }}

// 변경 후
style={{ width: `${artboardW}px`, height: `${artboardH}px` }}
```

> ⚠️ **중요**: `artboardW`/`artboardH`는 `elevationImages` 블록 내부에서 계산되므로,
> 아트보드 div에서 이 값을 사용하려면 계산 블록의 위치를 아트보드 div 바깥(상위)으로
> 끌어올리거나, 아트보드 div를 계산 블록 안으로 이동해야 합니다.
> 기존 구조를 최소한으로 변경하는 방법은 아트보드 div를 계산 IIFE 블록 내부로
> 포함시키는 것입니다. (현재 그리드 div가 IIFE 내부에 있는 구조를 활용)

#### B-4. 패널 정의 배열 — 변경 없음

```typescript
// W, D, H가 동적으로 반영되므로 별도 수정 불필요
{ id: 'top',   area: 'top',   w: W, h: D, r: 0   },
{ id: 'front', area: 'front', w: W, h: H, r: 0   },
{ id: 'rear',  area: 'rear',  w: W, h: H, r: 180 },

{ id: 'right', area: 'right', w: D, h: H, r: -90 },
```
