# 작업지시서 V28: 전개도 레이아웃 및 아트보드 개선

## 1. 개요 (Overview)
본 작업은 "Unfolded Box" 전개도 레이아웃에서 발생하는 이미지 잘림 현상을 해결하고, 건물의 비례(Building Ratio)가 클 때 아트보드 범위를 벗어나는 문제를 개선하기 위함입니다.

## 2. 주요 문제 분석 (Analysis)
### 2.1 이미지 잘림 현상 (Image Clipping)
- **원인**: 개별 이미지 컨테이너(`rotate-wrap` 하위 요소)에 `overflow: hidden`이 적용되어 있으며, `object-fit: cover` 사용으로 인해 생성된 이미지의 가장자리가 손실됨. 또한, 회전(Rotate) 시 컨테이너 크기 계산의 미세한 차이로 인해 경계선이 잘려 보일 수 있음.
- **추적 결과**: `src/App.tsx` 내 전개도 렌더링 루프(`map` 부분)의 CSS 클래스 및 인라인 스타일에서 `overflow-hidden` 속성이 보이지 않는 외곽선 역할을 하고 있음.

### 2.2 아트보드 이탈 및 가시성 저하 (Artboard Fit & Visibility)
- **원인**: 아트보드(`Library Artboard`)는 `1600x1200px`로 넉넉하지만, 내부 스케일링 계산 시 `1400x900px`이라는 과도하게 보수적인 가용 영역(Safe Area)을 사용하여 불필요한 여백이 발생함. 또한 `60px`의 넓은 간격(Gap)이 이미지 공간을 압박함.
- **실측 데이터 (H=15 기준)**:
    - **개별 이미지 컨테이너**: 약 `205px` x `308px` (아트보드 높이의 25% 수준)
    - **전체 그리드 점유율**: 가로 약 58%, 세로 약 75%로 여백이 과다함.
- **추적 결과**: `src/App.tsx`의 `availableW/H` 변수와 `gap-[60px]` 속성이 결합되어 이미지를 실질보다 작게 만들고 있음.

## 3. 통합 개선 제안 (Integrated Proposal)
가시성 확보와 아트보드 적합성을 동시에 만족시키기 위한 통합 솔루션입니다.

1. **가용 영역 극대화 (Safe Area expansion)**: `1400x900` → `1500x1100`으로 상향하여 아트보드 전체를 활용.
2. **그리드 간격 최적화 (Gap narrowing)**: 시각적 분리감은 유지하되 간격을 `60px` → **`12px`**로 대폭 축소하여 이미지 면적 확보.
3. **적응형 최소 스케일 (Adaptive Scale Floor)**: 배율(SCALE)이 너무 작아지지 않도록 하한선(예: 25px)을 설정하고, 초과 시 스크롤을 허용하여 디테일 보존.
4. **이미지 채우기 방식 변경**: `object-contain` 대신 비율에 맞게 꽉 채우는 `object-cover`와 정밀 비율 연산의 조합.

## 4. 수행 계획 (Execution Plan) - 업데이트
1. `src/App.tsx`에서 `availableW/H` 및 `gap` 수치 조정.
2. `SCALE` 계산 로직을 개선하여 아트보드 공간 점유율 90% 이상 목표.
3. `overflow-hidden` 적용 상태 유지 및 라벨 배치 미세 조정.

## 5. 검증 (Verification)
- 세로로 긴 건물(Tall building)과 가로로 긴 건물(Wide building) 모두 전개도가 아트보드 내에 안정적으로 렌더링되는지 확인.
- 회전된 뷰(Left, Right 등)의 이미지 모서리가 완벽하게 표현되는지 확인.
