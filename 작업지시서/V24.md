# 작업지시서 V24 (보고서)

**작성일:** 2026-04-03
**목적:** `backup_2026-04-03_v16_final` 상태로의 전체 시스템 롤백(Rollback) 계획 수립

---

## 1. 롤백 대상 분석 (`v16_final` 기준)

`backup_2026-04-03_v16_final` 폴더의 내용을 확인한 결과, 현재의 V23 상태에서 V16으로 되돌릴 경우 다음과 같은 **기능적/구조적 변화(Loss)**가 발생합니다.

### ① 주요 롤백 항목 (제거되는 기능들):
- **V17~V22 각도/방위 교정 로직:** 조감도(04:30/07:30) 시점의 좌/우 면 뒤바뀜 방어 시퀀스 및 사전 데이터가 사라집니다.
- **V18/V23 레이아웃 변경:** 현재의 3x3 혹은 2x4 T-Shape 레이아웃이 V16 당시의 레이아웃(초기 십자형 혹은 고전 그리드)으로 복구됩니다.
- **V19 스케치-이미지 동기화:** 이미지를 늘릴 때 스케치가 함께 늘어나는 비율 계산 로직(`scaleRatio`)이 제거됩니다.
- **V20 UI 최적화:** 단일 패널 사이드바, 'VIEWPOINT' 외부 라벨, 페이지 넘김 버튼 제거 등의 UX 개선 사항이 이전 상태(다중 탭/DNA 리포트 노출)로 되돌아갑니다.
- **V21~V22 시스템 제약 조건:** 투명 배경 강제(`OVERRIDE 1`), 비례 강제 동기화(`OVERRIDE 2`), 그리드 오버플로우 방어 코드(`minWidth:0`)가 모두 삭제됩니다.

---

## 2. 롤백 수행 계획 (Hard Reset)

롤백은 현재의 불안정하거나 복잡해진 상태를 초기 안정화 지점(V16)으로 완전히 되돌리는 과정입니다.

1.  **현 상태 백업 (Safety First):** 롤백 진행 전, 현재의 V23 상태를 `backup_2026-04-03_v23_pre_rollback`으로 백업하여 V17~V23의 작업 내용이 영구 손실되지 않도록 보호합니다.
2.  **루트 디렉토리 정리:** 현재 프로젝트 루트(`cai-change-viewpoint/`)의 파일들을 정리합니다.
3.  **V16 복원:** `backup_2026-04-03_v16_final` 디렉토리의 모든 파일과 폴더를 프로젝트 루트로 복사하여 덮어씁니다.
4.  **환경 정합성 확인:** V16 상태의 프로젝트에서 `npm install` 및 `npm run dev`를 실행하여 서버가 정상 가동되는지 확인합니다.

## 3. 요약 및 승인 요청

본 롤백은 **"최근 7회 분량의 업데이트(V17~V23)를 모두 무효화"**하고 V16의 순수 상태로 돌아가는 작업입니다. 

- **주의사항:** 롤백 후에는 최근에 해결했던 04:30/07:30 뷰 오류 등이 다시 발생할 수 있으며, 이를 V16 기반에서 다시 해결해야 할 수도 있습니다.

위 롤백 계획을 승인해 주시면 즉시 안전 백업 후 복원 작업을 진행하겠습니다.
