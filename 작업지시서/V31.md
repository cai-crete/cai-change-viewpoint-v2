# 작업지시서 V31

**작성일:** 2026-04-03
**목적:** 전개도(Unfolded Box) 내 5개 뷰 컨테이너의 크기(비율)를 사용자 요청 규격으로 고정 및 최적화

---

## 1. 요청 사항 분석
사용자께서 개별 이미지 컨테이너의 비율을 다음과 같이 변경 요청하셨습니다.

- **FRONT (정면)**: (W:10 x H:15) → **(W:12 x H:15)**
- **REAR (배면)**: (W:10 x H:15) → **(W:12 x H:15)**
- **LEFT (좌측면)**: (D:8 x H:15) → **(W:12 x H:15)** (D 성분을 12로 확장)
- **RIGHT (우측면)**: (D:8 x H:15) → **(W:12 x H:15)** (D 성분을 12로 확장)
- **TOP (평면)**: (W:10 x D:8) → **(W:12 x H:12)** (정사각형 비율로 확장)

**[분석 결과]**: 
모든 입면(Elevation)의 가로 길이를 **12**, 높이를 **15**로 통일하고, 평면(Top)의 경우 **12x12** 정사각형으로 규격화하는 것을 의미합니다. 이는 건축물의 실제 비율(`bldg_ratio`)과 별개로 **시트 상의 슬롯 크기를 표준화**하여 시각적 통일성을 확보하려는 의도로 파악됩니다.

---

## 2. 변경 계획 (Technical Plan)

### A. 파라미터 상수화
`App.tsx`의 `UnfoldedBox` 계산 로직에서 `bldg_ratio`를 그대로 사용하는 대신, 요청하신 규격을 우선 적용하도록 상수를 재정의합니다.

- `targetW = 12`
- `targetD = 12`
- `targetH = 15`

### B. 레이아웃 엔진 업데이트
이 변경에 따라 전체 레이아웃의 `SCALE` 계산 방식이 다음과 같이 업데이트됩니다.
- **수평 필요 공간**: `2 * targetH + targetW` = 30 + 12 = **42 유닛**
- **수직 필요 공간**: `2 * targetH + targetD` = 30 + 12 = **42 유닛**
- **동적 스케일**: `1476 / 42` (가로) 및 `1076 / 42` (세로) 중 최솟값 적용.

### C. 컨테이너 렌더링
- 각 슬롯(`front`, `rear`, `left`, `right`, `top`)은 위 상수를 기반으로 생성된 크기를 가집니다.
- `object-fit: cover` 속성이 이미 적용되어 있으므로, 실제 생성된 이미지가 해당 비율에 맞춰 꽉 차게 표시됩니다.

---

## 3. 질문 및 제안
1. **건축물 비율 동기화**: 이 변경은 "시각적 슬롯"의 크기만 바꾸는 것입니까, 아니면 AI 분석 시 반환되는 `bldg_ratio` 자체를 이 값으로 고정하는 것입니까? (제안: 시각적 가독성을 위해 슬롯 크기만 고정 적용하는 방향으로 진행하되, `bldg_ratio` 분석값은 유지하는 것이 데이터 무결성에 좋습니다.)
2. **배율 하락 가능성**: 유닛 수가 커짐에 따라(기존 가로 40 -> 42), 개별 이미지의 출력 픽셀 크기가 미세하게 줄어들 수 있습니다. (약 5% 내외)

**위 내용 확인 후 승인 주시면 즉시 적용하겠습니다.**
