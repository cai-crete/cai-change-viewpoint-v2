# 작업지시서 V30: 코너 뷰 각도 식별 오류(Flip) 최종 해결 (통합 제안)

## 1. 문제 현상 및 원인 분석
- **현상**: `V17`에서 주입한 [ANGLE CLOCK-FACE DICTIONARY]가 존재함에도 불구하고, AI가 `04:30`과 `07:30` 등을 여전히 반대로 인식하는 '각도 뒤집힘(Flip)' 현상이 보고됨.
- **원인 분석**: 
    1. **추론 순서의 부재**: AI가 이미지에서 왼쪽/오른쪽 입면을 먼저 확정하지 않고 각도를 먼저 찍은 뒤 사전에 끼워 맞추는 경향이 있음.
    2. **정면(Front) 식별 모호함**: 이미지 내의 어떤 면이 '정면'인지에 대한 시각적 근거가 부족할 때 AI가 임의로 판단함.
    3. **프롬프트 강제성 부족**: 사전을 참고하는 대신 자신의 '직관'을 우선시하는 AI 모델의 특성이 작용함.

## 2. 통합 해결 방안 (Strategic Prompt Overhaul)
단순 사전을 넘어, AI의 사고 과정을 논리적으로 강제하는 **'통합 추론 프로토콜'**을 적용합니다.

### ① 단계별 논리 추론(Chain-of-Thought) 강제
- AI가 각도를 내뱉기 전, 반드시 **"왼쪽 입면은 무엇인가?"**, **"오른쪽 입면은 무엇인가?"**를 먼저 서술하도록 JSON 구조를 변경합니다.
- 사전(Dictionary)과의 매칭 과정을 텍스트로 명시하게 하여 논리적 모순을 스스로 감지하게 합니다.

### ② 시각적 특징(Visual Anchors) 정의 추가
- **FRONT (정면)**: 건물의 주 출입구, 대형 로비, 주요 간판이나 디자인 포인트가 집중된 면으로 정의.
- **SIDE (측면)**: 창문 패턴이 반복되거나 부출입구가 있는 면으로 정의.
- 이 기준을 프롬프트에 추가하여 '정면'을 찾는 기준을 명확히 합니다.

### ③ 각도 판별 로직의 상향 (Logic Enforcement)
- "이미지의 왼쪽이 A이고 오른쪽이 B이면 각도는 무조건 C이다"라는 **IF-THEN 조건문** 형식으로 사전을 재구성하여 해석의 여지를 차단합니다.

## 3. 수행 계획 (Execution Plan)
1. `src/App.tsx`의 `analysisPrompt`를 '통합 추론 프로토콜' 기반으로 전면 개편.
2. `OUTPUT JSON SCHEMA`에 `visual_reasoning` (왼쪽/오른쪽 면 식별 및 사전 매칭 근거) 필드 추가.
3. 수정된 프롬프트로 `04:30`(정면-우측) 및 `07:30`(정면-좌측) 샘플 테스트 수행.
