## 1. 현황 분석

### 1-A. 이미지 치우침 — 크롭 후 건물 콘텐츠 미중앙 문제

#### 전제 (정상 동작)

```
• LEFT/RIGHT 컨테이너 회전(90°/-90°)은 올바름 (전개도 언폴드 컨벤션)
• cropCell() 크롭 좌표도 정확 (col/row 매핑 정상)
```

#### 발생 구조

```
AI 프롬프트 (line 977):
  "Upright = Ground side of each elevation MUST be at the bottom
   of its respective 3x3 cell."
  → 건물 바닥이 각 1/3 셀 하단에 위치
  → 투명 배경(sky)이 셀 상단을 채움

cropCell() 결과 (정사각형 이미지):
  ┌──────────┐
  │ (투명)   │  ← 셀 상단: 하늘/빈 공간
  │ (투명)   │
  ├──────────┤
  │ [건물]   │  ← 셀 하단: 건물 콘텐츠
  └──────────┘

object-contain in 컨테이너 (D*SCALE × H*SCALE portrait):
  → 정사각형 이미지를 가로(D*SCALE)에 맞춤
  → 이미지: D*SCALE × D*SCALE, 컨테이너 중앙 배치
  → 건물 콘텐츠: 이미지 하단에 집중

rotate(90°) 적용 후 (LEFT 패널):
  이미지 하단 → 패널 시각적 좌측
  → 건물이 패널 좌측으로 치우침 ✗

rotate(-90°) 적용 후 (RIGHT 패널):
  이미지 하단 → 패널 시각적 우측
  → 건물이 패널 우측으로 치우침 ✗
```

#### 영향 범위

| 패널 | 회전 | 치우침 방향 |
|---|---|---|
| FRONT | r=0 | 건물이 컨테이너 하단 |
| REAR | r=180 | 건물이 컨테이너 상단 |
| LEFT | r=90 | 건물이 패널 시각적 좌측 ✗ |
| RIGHT | r=-90 | 건물이 패널 시각적 우측 ✗ |
| TOP | r=0 | 건물이 컨테이너 중앙 (상대적 무해) |

→ LEFT/RIGHT가 가장 심각 (회전으로 인해 치우침이 시각적으로 크게 부각됨)

## 2. 변경 계획 (V35 Technical Plan)

### A. cropCell → trimAndCenter 적용 (line 1020–1032)

#### 원리

```
크롭 후 → 투명 픽셀 경계 감지 → 건물 바운딩박스 추출
       → 새 정사각형 캔버스에 건물 콘텐츠를 중앙 배치
       → 이 재중앙화된 이미지를 elevationImages에 저장

결과:
  ┌──────────┐
  │ (투명)   │  ← 균일한 패딩
  │ [건물]   │  ← 중앙
  │ (투명)   │  ← 균일한 패딩
  └──────────┘

object-contain + rotate → 건물이 패널 중앙에 위치 ✓
```

#### 추가 함수 (cropCell 바로 위에 삽입)

```typescript
// [V35] 크롭된 셀의 투명 영역을 제거하고 건물 콘텐츠를 중앙에 재배치
const trimAndCenter = (canvas: HTMLCanvasElement): string => {
  const ctx = canvas.getContext('2d');
  if (!ctx) return canvas.toDataURL();
  const { width, height } = canvas;
  const data = ctx.getImageData(0, 0, width, height).data;

  let minX = width, maxX = 0, minY = height, maxY = 0;
  let hasContent = false;
  for (let y = 0; y < height; y++) {
    for (let x = 0; x < width; x++) {
      if (data[(y * width + x) * 4 + 3] > 10) {
        if (x < minX) minX = x;
        if (x > maxX) maxX = x;
        if (y < minY) minY = y;
        if (y > maxY) maxY = y;
        hasContent = true;
      }
    }
  }
  if (!hasContent) return canvas.toDataURL(); // fallback: 투명 없으면 원본 반환

  const bldW = maxX - minX + 1;
  const bldH = maxY - minY + 1;
  const innerSize = Math.max(bldW, bldH);
  const padding = Math.round(innerSize * 0.08); // 8% 여백
  const totalSize = innerSize + padding * 2;

  const out = document.createElement('canvas');
  out.width = totalSize;
  out.height = totalSize;
  const outCtx = out.getContext('2d')!;
  const destX = padding + Math.round((innerSize - bldW) / 2);
  const destY = padding + Math.round((innerSize - bldH) / 2);
  outCtx.drawImage(canvas, minX, minY, bldW, bldH, destX, destY, bldW, bldH);
  return out.toDataURL();
};
```

#### Fallback 처리

```
trimAndCenter 내부:
  • 비투명 픽셀이 없는 경우(empty cell) → 원본 canvas.toDataURL() 반환 (안전)
  • 이미지가 불투명 배경(JPEG 등)인 경우 → 전체 이미지가 hasContent=true → 트림 없음
    → 원본과 동일한 결과 반환 (안전)
```
