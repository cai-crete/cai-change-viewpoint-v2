## 1. 현황 분석

현재 `analyzeViewpoint`의 프롬프트는 `1_Geometry_MASTER`와 `2_Property_SLAVE`를 단순 **키-값(key-value) 단일 블록**으로만 요청하고 있습니다.

**누락된 핵심 구조:**
- `§2. 입면별 구체적 작성 기준` → 정면·지붕·우측·좌측·배면 **뷰별 독립 서술** 없음
- `§3. 입면 정의 마스터 템플릿` → `[Meta Data]`, `Part A (A-1 ~ A-5)`, `Part B (B-1 ~ B-5)` **5면 × 2파트 구조** 없음
- 개발자 로그도 뷰 단위 구조 미출력

---

## 2. 수행 계획

### 2-1. AI 프롬프트 업그레이드 (`analysisPrompt`)
`analysisPrompt` JSON 스키마를 5면 독립 뷰 구조로 전면 재설계합니다.

```
elevation_views: {
  Front: { meta: {...}, "1_Geometry_MASTER": { A-1 ~ A-5 }, "2_Property_SLAVE": { B-1 ~ B-5 } },
  Top:   { ... },
  Right: { ... },
  Left:  { ... },
  Rear:  { ... }
}
```

### 2-2. PHASE 2 진입 잠금 (게이트 검증)
AI 응답에서 `elevation_views`의 5개 뷰가 모두 존재하고, `1_Geometry_MASTER`와 `2_Property_SLAVE`가 각 뷰에 성립한 경우에만 `generateSitePlan()` (PHASE 2) 진입을 허가합니다.  
검증 실패 시 알림 후 PHASE 2 차단.

### 2-3. 개발자 콘솔 로그 확장
개발자 콘솔에 5면 각각의 `Geometry_MASTER`와 `Property_SLAVE`를 **뷰 단위로 구분하여 출력**합니다.

---

## 3. 불확실한 정보 — 1개 질문

> [!IMPORTANT]
> PHASE 2 게이트 검증 실패 시 처리 방식에 대해 결정이 필요합니다.

**제안 1. 경고 후 통과 허용 (Soft Gate)**
- 5면 데이터가 불완전하더라도 콘솔 경고만 띄우고 PHASE 2를 계속 진행합니다.
- 이미지를 잘 인식하지 못한 경우에도 흐름이 끊기지 않아 사용자 경험이 좋습니다.

**제안 2. 경고 팝업 후 통과 허용 (회색 배너 UI 경고)**
- 5면 데이터 누락 시 화면 상단에 `"⚠️ 일부 입면 정보가 누락되었습니다. 결과가 부정확할 수 있습니다."` 배너를 4초간 표시하고 계속 진행합니다.

**제안 3. 완전 차단 (Hard Gate)**
- 5면 데이터가 모두 충족되지 않으면 PHASE 2 진입 자체를 막고, 사용자에게 재분석을 요청하는 모달을 표시합니다.
