# 작업지시서 V1

**작성일:** 2026-04-02
**목적:** Vercel 배포 문제 해결 및 기본 환경 설정

## 현재 상태 분석
Vercel 배포 후 루트 경로에서 404 Not Found 오류가 발생하고 있습니다. 이는 SPA(Single Page Application) 라우팅 설정이 누락되었거나 Vercel이 프로젝트 빌드 결과물(dist)을 올바르게 인식하지 못해 발생할 가능성이 높습니다.

## 주요 작업 내용
1. **API 키 설정:** `.env` 파일에 `GEMINI_API_KEY`를 추가하여 AI 모델 구동 환경을 구축하였습니다.
2. **리포지토리 초기화:** Git을 초기화하고 최신 코드를 GitHub 리포지토리에 푸시하였습니다.
3. **Vercel 설정 최적화:** `vercel.json` 파일을 추가하여 라우팅 문제를 해결하고 빌드 출력 경로를 명시합니다.
4. **API 키 호출 오류 분석 (추가):**
    - **원인:** `.env` 파일은 보안상 GitHub에 업로드되지 않으므로, Vercel 서버에서는 해당 키를 찾을 수 없습니다. 또한 Vite 환경에서는 브라우저에 노출될 변수에 `VITE_` 접두사가 필요합니다.
    - **해결 계획:**
        - `App.tsx` 코드 내 환경 변수 호출 방식을 하나로 통일합니다.
        - `vite.config.ts` 설정을 보완하여 `VITE_` 접두사 유무에 상관없이 키를 인식하게 합니다.
        - 사용자가 Vercel 대시보드(Settings > Environment Variables)에서 직접 `VITE_GEMINI_API_KEY`를 등록해야 합니다.

## 향후 계획
- Vercel 대시보드에 API 키 등록 안내 및 코드 수정.
- Vercel 재배포를 통한 서비스 정상화 확인.
- 추가 기능 구현 및 UI 개선 진행.
