# 작업지시서 V16

**작성일:** 2026-04-03
**목적:** "업로드 원본의 전개도 비율 오류 및 생성된 이미지 전개도 비율 정상화" 현상 원인 분석 및 해결 방안

---

## 1. 문제 현상 및 원인 분석

첨부해주신 스크린샷과 코드를 교차 검증한 결과, **정확하게 반대로 동작하여 발생한 착시 현상**입니다.
원본 이미지의 아트보드가 비정상인 것이 아니라 코딩된 5패널 분리 그리드가 비정상적으로 왜곡되어 출력된 것이며, 생성된 이미지 쪽이 정상인 이유는 우연히 예전 구형 방식(통짜 이미지 출력)으로 표시됐기 때문입니다.

### [원인 1] 분석 직후 원본 이미지의 "비율 왜곡 버그"
- `App.tsx`의 V11 업데이트 코드를 보면 `analyzeViewpoint()`에서 AI가 건축물 실제 비례(`data.bldg_ratio`)를 정상 추출하여 원본 아이템에 저장합니다.
- 그런데 바로 다음 실행되는 **`generateSitePlan()` 함수 내부에서 `bldgRatio: null` 로 강제 덮어쓰기 하는 실수 코드**가 남아 있었습니다.
- 그 결과, 원본 이미지의 5면 CSS Grid가 사용자의 건물 실제 비례를 잃어버리고, 시스템의 Fallback 기본값(`width: 10, depth: 7, height: 13`)을 강제로 사용하게 되어 비율이 완전히 왜곡(길쭉하게 찌그러짐)되어버렸습니다.

### [원인 2] Generate 된 이미지의 "비율 정상화 착시 현상"
- `handleGenerate()` 함수가 시점 변경된 렌더링 신규 이미지를 캔버스에 추가할 때, **`elevationImages(5면 크롭 데이터)`와 `bldgRatio` 변수를 자식 아이템에게 넘겨주지 누락하는 버그**가 있었습니다.
- 자식 아이템(GENERATE 이후)은 5면 크롭 데이터가 없으므로 V11형 CSS 비례 그리드 자체를 렌더링하지 못하고, 구형 V10의 단일 `architecturalSheetImage`(통짜 이미지)를 화면에 띄웁니다.
- 통짜 이미지는 AI가 렌더링한 원본 비율 그대로이므로 **"역설적으로 정상적인 비율"**처럼 보였던 것입니다.

## 2. 해결 방안 (수행 계획)

이 문제는 두 가지 누락된 데이터 파이프라인을 연결하면 완벽히 해결됩니다.

### 조치 1: `generateSitePlan`의 `null` 덮어쓰기 제거
- `generateSitePlan`에서 원본 아이템 업데이트 시 `bldgRatio: null` 부분 삭제.
- 이를 통해 원본 이미지를 클릭했을 때 V11 기반 5패널 분리형 CSS Grid가 AI가 추출한 `bldg_ratio`에 맞게 무결점 스웹 비율로 출력되도록 수정.

### 조치 2: `handleGenerate` 데이터 상속 누락 수정
- Generate 후 파생된 자식 아이템을 생성할 때, 부모(원본) 아이템이 가진 `elevationImages`와 `bldgRatio`를 자식의 `parameters`에도 그대로 복사하여 상속시켜 줍니다.
- 이를 통해 자식 이미지에서도 구형 단일 시트가 아니라, 정상 비율이 맞춰진 5패널 CSS Grid가 작동하도록 버그 픽스.
