## 1. 현재 방위 로직 분석

시스템이 정의한 건축 방위 좌표계 상수(06:00=앞, 03:00=우, 09:00=좌, 12:00=후)는 올바르게 설정되어 있습니다.

**각 시점에서 관찰할 때 2D 이미지의 화면 상에 나타나는 좌/우 관계:**
- **04:30 (Front-Right 코너):** 빨간색(우전) 모서리가 중앙에 옴. 모서리 기준 **왼쪽** 화면은 `FRONT`, **오른쪽** 화면은 `RIGHT`.
- **07:30 (Front-Left 코너):** 파란색(좌전) 모서리가 중앙에 옴. 모서리 기준 **왼쪽** 화면은 `LEFT`, **오른쪽** 화면은 `FRONT`.

## 2. 사선 뷰 방향 오류(좌우 반전)의 근본 원인 파악

현재 `App.tsx`의 `handleGenerate`에서 코너 뷰(04:30, 07:30 등)를 생성할 때 2장의 도면 이미지(IMAGE 2, IMAGE 3)를 전달하는 프롬프트는 다음과 같습니다.

```text
- IMAGE 2 (Geometric Reference A): The pre-computed RIGHT elevation...
- IMAGE 3 (Geometric Reference B): The pre-computed FRONT elevation...
  Both elevations are pre-computed by PHASE 2. Use them as the STRICT geometric blueprint.
```

**[오류 원인]**
AI에게 2장의 도면을 던져주고 "이 두 개를 결합해서 2소실점 투시도를 만들어라"라고 지시했지만, **"어떤 도면이 결과 이미지의 화면 왼쪽 면이고 어떤 도면이 오른쪽 면인지(좌우 배치 규칙)"**를 전혀 명시하지 않았습니다.
결과적으로 AI는 확률적으로 도면을 좌우 무작위로 붙이게 되며, 이로 인해 `Front`를 뒷면으로 보내버리거나 좌우를 반전(Flip)시키는 심각한 기하학적 오류가 종종 발생합니다.

---

## 3. 해결 방안 (3가지 제안)

### 제안 1. 코너 면의 프레임 내 좌/우 위치 명시 (가장 확실한 해결책) ★ 추천
프롬프트에 화면 상의 **Left-hand side wall / Right-hand side wall** 배치를 명시적으로 강제합니다.
```javascript
// 04:30 예시
- IMAGE 2 (FRONT): 이 입면은 결과 이미지 렌더링 시 반드시 모서리 기준 **왼쪽 화면(Left-hand side)**에 위치해야 한다.
- IMAGE 3 (RIGHT): 이 입면은 결과 이미지 렌더링 시 반드시 모서리 기준 **오른쪽 화면(Right-hand side)**에 위치해야 한다.
```
- **효과:** 시각적 구속력이 명확해져 AI의 좌우 반전 실수를 100% 차단합니다. 구현도 간단합니다.

### 제안 2. 모서리(Edge) 앵커 포인트 지시
프롬프트에 "IMAGE 2의 오른쪽 끝선과 IMAGE 3의 왼쪽 끝선을 맞대어 중앙의 Red Edge(우전 모서리)를 형성하라"는 식으로 Vertex(꼭짓점) 결합 방식을 수학적으로 서술합니다.
- **효과:** 원리는 정확하지만 AI가 평면 도면의 Vertex 결합을 텍스트로 이해하기 다소 어려울 수 있습니다.

### 제안 3. 코너 생성용 단일 파노라마 이미지 제공 (PHASE 2 수정 필요)
코너 뷰를 렌더링할 때 도면을 2개로 분리해서 주지 않고, PHASE 2에서 이미 두 면이 올바른 순서로 붙어 있는 1장의 가로로 긴 이미지를 크롭하여 제공합니다.
- **효과:** 좌우 혼동 여지 차단. 다만 `generateSitePlan`의 구조를 또다시 변경해야 하는 부담.
