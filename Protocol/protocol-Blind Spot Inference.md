# protocol-Blind Spot Inference

# **System Protocol: Blind Spot Inference & Reconstruction**
**(사각지대 추론 및 재구성 프로토콜)**

## **I. Core Logic: Detection to Reconstruction (발견에서 재구성으로)**
이 시스템의 핵심 미션은 **"Detection to Reconstruction"**입니다. 입력된 이미지의 **가시권(Visible Zone)** 정보를 정밀 분석하여 **'Design DNA'**를 추출하고, 이를 바탕으로 보이지 않는 **사각지대(Blind Spot)**를 건축적 문법에 맞게 설계하여 완전한 건축물로 복원합니다.
* **원칙:** 외관을 단서로 내부 프로그램(용도)을 유추하고, 이에 따른 **코어(Core: 계단, 화장실, 설비)**의 위치를 확정하는 **'Inside-Out Logic'**을 따릅니다.
* **전략:** 가시권의 DNA를 사각지대로 **확장(Extrusion)**하되, 기능적 요구(설비, 서비스 동선)에 맞춰 디자인을 **변주(Variation)**합니다.
---

## **II. Phase 1: Context & Zoning Inference (맥락 및 조닝 추론)**
*적용 규칙: `{건축계획규칙}` (Architectural Planning Rules)*
건물의 배치 형태와 내부 기능을 추론하여 보이지 않는 면의 **'뼈대'**를 잡는 단계입니다.

### **1. Context Typing (대지 조건 유형화)**
입력 이미지의 배경 단서(도로, 인접 건물)를 분석하여 건물의 성격을 규정하고, 이에 따라 측/배면의 개방 정도를 결정합니다.
* **Type A [Urban/Attached]:** 도심 밀집형으로 판단될 경우, 인접 건물이 존재한다고 가정합니다.
* **Action:** 측면 및 배면은 폐쇄적이고 기능적인 **방화벽(Firewall)** 형태로 계획합니다.
* **Type B [Nature/Detached]:** 독립 랜드마크형으로 판단될 경우, 4면이 개방된 대지로 가정합니다.
* **Action:** 정면의 디자인 언어를 4면에 **연속적(Continuity)**으로 확장하여 조형성을 강조합니다.

### **2. Inside-Out Zoning (내부 기능 기반 입면 계획)**
* **Program Inference:** 외관(창호 크기, 층고)을 단서로 건물의 용도(주거, 오피스, 상업)를 추론합니다.
* **Virtual Core Placement:** 추론된 용도에 따라 **코어(Core: 계단실, 화장실, 엘리베이터)**와 설비 공간(MEP)의 위치를 **배면(Rear)** 또는 **측면 후방**에 배치합니다.
* *중요:* 이 코어의 위치는 배면 창호 디자인을 결정하는 결정적 근거가 됩니다.
---

## **III. Phase 2: Facade Generation Rules (입면 생성 규칙)**
*적용 규칙: `{입면생성규칙}` (Facade Generation Rules)*
추출된 DNA(재질, 스타일)와 수립된 조닝 계획을 결합하여, 각 사각지대(Rear, Side, Top)의 구체적인 디자인을 확정합니다.

### **1. Rear Facade Logic (배면 구성 로직)**

* **Form follows Function (형태는 기능을 따른다):** 정면의 디자인을 그대로 복사하지 않고, Phase 1에서 배치한 코어(Core)에 따라 **'기능적 변주'**를 수행합니다.
* **Detailing Strategy:**
* **Door:** 주 출입구 대신 **서비스 도어(Service Door)**를 배치합니다.
* **Window:** 대형 창호 대신 환기 위주의 **작은 창(Ventilation Window)**이나 **루버(Louver)**로 대체합니다.
* **MEP Details:** 리얼리티를 높이기 위해 설비 덕트, 실외기, 가스 배관 등 **설비적 디테일(MEP Details)**을 추가합니다.

### **2. Side Extrusion Logic (측면 연장 로직)**
* **Wrapping:** 정면의 마감재와 슬라브 라인이 측면으로 끊김 없이 이어지도록 **'디자인 래핑(Wrapping)'**을 수행합니다.
* **Density Control:**
* Type A(도심형)일 경우 개구부를 최소화합니다.
* Type B(독립형)일 경우 정면과 동일한 개방감을 부여합니다.

### **3. Top Definition (지붕 및 옥상 정의)**
* **Logic:** 파라펫(Parapet)의 유무와 형태를 단서로 지붕의 종류를 정의합니다.
* **Flat Roof (평지붕):** 방수 마감 텍스처를 적용하고, **HVAC(공조기), 물탱크, 태양광 패널** 등 설비 오브제를 배치합니다.
* **Pitched Roof (경사지붕):** 지붕재 패턴(징크, 기와 등)을 입히고, **빗물받이(Gutter) 및 선홈통(Downspout)** 시스템을 연결합니다.
---

## **IV. Phase 3: Verification (충돌 검증)**
* **Conflict Check:** Protocol A의 설계 의도(예: 배면의 설비 덕트 설치)가 Protocol B의 심미적 기준(이미지의 깔끔함)과 충돌하지 않도록 조율합니다.
* **Logical Consistency:** 전면 디자인이 측면과 배면으로 **논리적으로 연결**되었는지 최종 검증합니다.
