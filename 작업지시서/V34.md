## 1. 현황 분석 — 클리핑 2종

### 1-A. [주요] 컨테이너 내부 이미지 클리핑 (object-cover 문제)

#### 발생 구조

```
AI 시트 생성 방식:
  전체 이미지: N × N (정사각형, 예: 1024×1024px)
  3×3 등분할 → 각 셀: N/3 × N/3 = 341×341px (정사각형)

패널 표시 방식 (현재):
  컨테이너: W*SCALE × H*SCALE = 건축물 비율 반영 (비정사각형)
  이미지: 341×341px 정사각형
  object-fit: cover → 컨테이너 완전 채우기 위해 스케일 후 크롭
```

#### 클리핑 계산 예시 (W=8, D=6, H=20, SCALE=30)

| 패널 | 컨테이너 크기 | object-cover 기준 | 보이는 비율 | 클리핑 |
|---|---|---|---|---|
| FRONT/REAR | 240×600px (세로) | 높이(600) 기준 스케일 | 가로 40% | **좌우 60% 클리핑** |
| LEFT/RIGHT | 180×600px (세로) | 높이(600) 기준 스케일 | 가로 30% | **좌우 70% 클리핑** |
| TOP | 240×180px (가로) | 가로(240) 기준 스케일 | 세로 75% | **상하 25% 클리핑** |

→ 건물이 세로로 길수록 (H >> W, D) 클리핑 심화

#### 코드 위치

**파일**: `src/App.tsx`, line 1676

```typescript
<img src={ei[view.id]} className="w-full h-full object-cover block" alt={view.id} />
//                                                    ^^^^^^^^^^^
//                                                    컨테이너 밖 이미지 크롭 원인
```

### 1-B. [부차] 아트보드 경계 클리핑 (mt-10 초과 문제)

**코드 위치**: `src/App.tsx`, line 1647

```typescript
<div className="grid gap-[12px] relative mt-10"
//                                         ^^^^
//                                         40px 상단 마진 → 그리드가 콘텐츠 영역 40px 초과
```

```
artboardH 설계: 콘텐츠 영역 = gridH
실제 그리드:    gridH + mt-10 = gridH + 40px
→ 40px 초과 → 수직 스크롤바 → 스크롤바 폭으로 수평도 초과 → 양방향 스크롤바
→ FRONT 하단, REAR 상단 패널 경계 클리핑
```

## 2. 변경 계획 (V34 Technical Plan)

### A. [주요] object-cover → object-contain (line 1676)

```diff
- <img src={ei[view.id]} className="w-full h-full object-cover block" alt={view.id} />
+ <img src={ei[view.id]} className="w-full h-full object-contain block" alt={view.id} />
```

#### 변경 후 동작

```
object-contain: 이미지 비율 유지하며 컨테이너 내부에 완전 표시 (크롭 없음)

FRONT (240×600px 컨테이너, 341×341px 이미지):
  → 가로 기준 맞춤: 240×240px로 표시
  → 상하 여백: (600-240)/2 = 180px (패널 배경색 표시)
  → 건축물 입면 전체가 잘림 없이 표시 ✓

LEFT/CSS (180×600px 컨테이너, 341×341px 이미지):
  → 가로 기준 맞춤: 180×180px로 표시
  → 상하 여백 발생 (패널 배경색)
  → 건축물 측면 전체가 잘림 없이 표시 ✓
```

**여백 처리**: 현재 `bg-black/5`(≈ 연회색)로 채워짐 → 시각적으로 무해함
(향후 개선 시 여백을 흰색으로 통일 가능)

---

### B. [부차] artboardH에 TITLE_RESERVE 추가 (line ~1612)

```diff
  const artboardW = Math.ceil(unitsW * SCALE + gapTotal + PADDING);
- const artboardH = Math.ceil(unitsH * SCALE + gapTotal + PADDING);
+ const TITLE_RESERVE = 40; // grid의 mt-10(40px) 마진 보정
+ const artboardH = Math.ceil(unitsH * SCALE + gapTotal + PADDING + TITLE_RESERVE);
```

#### 변경 후 검증

```
새 콘텐츠 영역 높이 = artboardH − PADDING = gridH + 40
그리드 마진 박스    = gridH + mt-10 = gridH + 40
→ 콘텐츠 영역 = 마진 박스 → 초과 없음 → 스크롤바 없음 ✓

타이틀 위치 검증:
  타이틀 하단(82px) < REAR 패널 시작(120px) → 겹침 없음 ✓ (38px 여유)
```
