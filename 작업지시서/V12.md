# 작업지시서 V12

**작성일:** 2026-04-02
**목적:** PHASE 2(전개도 생성)와 PHASE 3(비가시권 생성)의 Blind Spot 추론 충돌 분석 및 해결 방안

---

## 1. 현재 구조 추적 (Blind Spot Inference)

### [PHASE 2] 5면 전개도 생성 (`generateSitePlan`)
- **목표:** 원본 이미지와 5-View 구조화 파라미터(AEPL)를 기반으로 5면(Front, Top, Rear, Left, Right)의 전개도를 한 장의 시트에 생성합니다.
- **주입 정보:** 구조화된 자연어 파라미터 (스케일, 그리드, 깊이, 개구부, 재료 등) 및 시점(Viewpoint) 정보.
- **산출물:** `elevationImages` (각 뷰의 크롭 이미지).

### [PHASE 3] 시점 변경 생성 (`handleGenerate`)
- **목표:** 사용자가 지정한 시점으로 이미지를 재생성.
- **주입 정보 1 (시각):** **IMAGE 2** — PHASE 2에서 생성된 형태 도면 (예: 12:00 시점이면 `Rear` 이미지 추출).
  - *프롬프트 지시:* "IMAGE 2를 STRICT geometric blueprint로 사용하여 형상과 비율을 완벽히 매핑하라."
- **주입 정보 2 (규칙):** **Layer C: Blind Spot Inference** — 비가시권일 경우 `protocol-Blind Spot Inference` 규칙 텍스트 주입.
  - *프롬프트 지시:* "서비스 도어(SERVICE DOOR) 생성, 환기용 작은 창문, MEP 디테일 (공조기, 배관) 추가, 코어 배치 등 반영."

---

## 2. 모순 및 충돌 관계 파악

PHASE 3에서 AI는 **"시각적 제한(IMAGE 2)"**과 **"텍스트 규칙(Layer C)"** 사이에서 충돌을 겪습니다.

- **원인:** PHASE 2(전개도 생성)의 프롬프트에는 `protocol-Blind Spot Inference`의 상세 규칙(서비스 도어, MEP, Type A/B 조닝 등)이 **주입되어 있지 않습니다.**
- **결과:** PHASE 2에서 생성된 Rear(배면) 이미지는 단순히 "창문이 뚫린 뒷벽"이나 "정면을 복제한 밋밋한 벽"으로 형상화됩니다.
- **충돌:** 
  1. AI는 텍스트(Layer C)를 읽고 "서비스 도어와 환기 배관을 그려야지"라고 판단합니다.
  2. 그러나 IMAGE 2(PHASE 2의 결과물)를 "STRICT geometric blueprint"로 따르라는 엄격한 지시 때문에, IMAGE 2에 없는 텍스트 규칙(서비스 도어 등)의 적용을 주저하게 됩니다.
  3. 결과적으로 **기하학적 제약과 텍스트 논리가 상충하여, 모호한 결과물이나 원본을 재출력하는 Hallucination**으로 이어질 가능성이 존재합니다.

---

## 3. 해결 방안 (3가지 제안)

이 충돌을 해결하기 위해 어느 쪽에 우선순위를 둘지(또는 어떻게 동기화할지) 결정해야 합니다.

### 제안 1. PHASE 2에 Blind Spot 세부 규칙 통합 (근본적 동기화) ★ 추천
PHASE 2의 `sitePlanPrompt` 생성 시점에도 `protocol-Blind Spot Inference` (서비스 도어, MEP 등 조닝 규칙) 텍스트를 주입합니다.
- **효과:** PHASE 2에서 생성되는 전개도(IMAGE 2) 자체가 이미 텍스트 규칙을 선반영하게 되며, PHASE 3에서는 형태(IMAGE 2)와 텍스트(Layer C)가 완벽히 일치하여 강력한 시너지를 냄.

### 제안 2. PHASE 3에서 기하학적 제약 완화 (Layer C 텍스트 우선)
PHASE 3 `handleGenerate` 프롬프트에서 IMAGE 2를 "STRICT geometric blueprint"로 따르라는 문구를 완화합니다.
*(예: "IMAGE 2는 전체적인 매스(Mass) 참고용이며, 세부 디테일(창문, 도어)은 Step 3 (Blind Spot 규칙)을 최우선으로 따른다.")*
- **효과:** 빠르게 충돌을 피할 수 있으나, PHASE 2 전개도와 최종 렌더링 간의 세부 불일치가 발생할 수 있음.

### 제안 3. 비가시권은 IMAGE 2 참조 제외 (순수 텍스트 생성)
비가시권(Rear, Side)을 렌더링할 때는 PHASE 2의 IMAGE 2(전개도)를 입력에서 아예 제외하고, Original Image(맥락) + Layer C(규칙) 만으로 생성하도록 로직 변경.
- **효과:** IMAGE 2로 인한 충돌은 원천 차단되지만, 건물의 전체 비율(Proportion) 일관성이 떨어질 수 있음.
